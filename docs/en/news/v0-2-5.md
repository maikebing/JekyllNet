JekyllNet `v0.2.5` is now released. This update packages every externally visible improvement since `v0.2.0` into a more complete release: broader theme coverage, stronger config compatibility, cleaner release defaults, and a more consistent GitHub Pages workflow story.

## Official links

- Repository: [https://github.com/JekyllNet/JekyllNet](https://github.com/JekyllNet/JekyllNet)
- Website: [https://jekyllnet.help](https://jekyllnet.help)
- GitHub Pages entry: [https://jekyllnet.github.io/JekyllNet/](https://jekyllnet.github.io/JekyllNet/)
- Action: [https://github.com/JekyllNet/action/tree/v2.5](https://github.com/JekyllNet/action/tree/v2.5)

## Highlights in v0.2.5

- Added `jekyll-theme-lumen` to the tested theme set
- Improved theme and plugin support across the rendering pipeline
- Added `{{version}}` placeholder resolution after YAML config loading for themes such as al-folio
- Updated theme submodules and Pages workflow references to keep examples and matrix builds aligned
- Moved the default GitHub Action tool install target to `JekyllNet 0.2.5`

## What changed for users

- Theme compatibility coverage is broader and more realistic for modern GitHub Pages style sites
- Action examples and reusable workflows now target `JekyllNet/action@v2.5`
- Release defaults are easier to understand because the Action tag and default CLI version now move together
- Docs, news, and site navigation now point to the current release instead of the older `v0.2.0` announcement
