今宣布，JekyllNet `v0.2.5` 正式发布。本次版本并非只增数项零散修补，而是将自 `v0.2.0` 以来所有对外可感知之改动收束为一轮完整交付：主题兼容面更广、配置兼容性更强、发布默认值更清晰、GitHub Pages 工作流链路更统一。

## 官方入口

- [JekyllNet 仓库](https://github.com/JekyllNet/JekyllNet)
- [JekyllNet 文档网站](https://jekyllnet.help)
- [JekyllNet GitHub Pages 入口](https://jekyllnet.github.io/JekyllNet/)
- [JekyllNet Action v2.5](https://github.com/JekyllNet/action/tree/v2.5)

## v0.2.5 核心亮点

- 新增 `jekyll-theme-lumen`，纳入已测试主题范围
- 增强主题支持与插件架构，提高跨主题渲染与构建稳定性
- `_config.yml` 载入后新增 `{{version}}` 占位符解析，兼容如 al-folio 等主题的版本变量写法
- 更新多个主题子模块与 Pages 工作流引用，令示例、矩阵构建与文档叙述相一致
- GitHub Action 默认安装版本提升到 `JekyllNet 0.2.5`

## 对使用者最直接的变化

- 多主题站点的兼容范围更广，主题验证更贴近真实使用场景
- 文档示例与可复用工作流统一切换到 `JekyllNet/action@v2.5`
- Action 标签与默认 CLI 版本同步推进，降低 CI 配置理解成本
- 首页、导航、新闻与发布说明均已指向当前版本，而不再停留在旧版发布稿
